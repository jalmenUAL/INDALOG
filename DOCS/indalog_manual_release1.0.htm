INSTALLING INDALOG

INDALOG has been developed under SWI-PROLOG, therefore firstly install SWI-PROLOG in your computer.  Secondly, uncompress the distribution. The distribution is organized in the following directories:

BIN

EXAMPLES-PROGRAMS

EXAMPLES-QUERIES

DOCS

BIN includes the file “indalog.pl” which is the root file of the distribution. Therefore you have to load the file from the SWI-Prolog command line:

?- [indalog].

% indalog compiled 0.03 sec, 4,228 bytes

LOADING PROGRAMS

Now, you can test the examples in directory EXAMPLES-PROGRAMS. There are two examples: “employer.pl” and “points.pl”.  They includes two functional-logic databases. Databases in Indalog are declared as follows:

database employer(name,address,number,phone).

database employer_age(name,age).

key employer(name).

key employer_age(name).

datatype addresstype ::= add(number_st,flattype).

datatype flattype ::= flat(number_fl,letter).

datatype letter ::= a | b | c | d | e.

datatype person ::= peter  | mark | frank.

datatype number_st ::= first | second | third | fourth | fifth.

datatype number_fl ::= '1' | '2' | '3' | '4' | '5'.

employer :: person -> ok.

employer(peter):=ok.

employer(mark):=ok.

employer(frank):=ok.

employer_age :: person -> ok.

employer_age(peter):=ok.

employer_age(mark):=ok.

employer_age(frank):=ok.

address :: person -> addresstype.

address(peter):=add(fifth,flat(4,a)).

address(mark):=add(second,flat(5,b)).

address(frank):=add(first,flat(3,c)).

age :: person -> number.

age(peter):=age(mark).

age(mark):=19.

age(frank):=29.

number :: person -> number.

number(frank):=29.

number(peter):=14.

number(mark):=8.

phone :: person -> number.

phone(peter):=238987.

phone(mark):=123456.

phone(frank):=198976.

bonus :: person -> number.

bonus(peter):=1.

bonus(frank):=2.

To load the examples you have to use the Indalog command “ld”:

?- ld employer.

% employer compiled 0.00 sec, 532 bytes

Or

?- ld points.

% points compiled 0.00 sec, 4,488 bytes

EXECUTING QUERIES

Now, you can make queries included in the file “functional.pl” by using the Indalog command “evq”.

For testing SQL-style queries in the file “sql.pl” you have to use the Indalog command “evs”.

We hope the examples will run in your computer.
